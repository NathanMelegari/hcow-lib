# Contributing with Hcow

Thanks for your interest in contributing to Hcow!
Please follow the guidelines below to ensure a smooth contribution process.

---

## How to Contributing

### 1. Fork the repository

Click the "Fork" button and clone your fork:

```bash
git clone https://github.com/your-username/hcow-lib.git
```

### 2. Create a new branch

```bash
git checkout -b feature/your-feature-name
```

### 3. Make your changes
- Add new features or improve existing ones
- Keep the code clean and readable
- follow the existing code style

### 4. Build and test

```bash
mkdir build
cd build
cmake ..
make
```

### 5. Commit your changes

```bash
git commit -m "feat: add new feature"
```

### 6. Push your branch

```bash
git push origin feature/your-feature-name
```

### 7. Open a pull request

Go to the original repository and open a Pull Request.
Explain clearly what you changed and why.

## Guidelines

- Keep the API clean and consistent (e.g., hcow_run, hcow_log)
- Write clear and maintainable code
- Add comments when necessary
- Avoid unnecessary dependencies

## Suggestions for Contributing

- New automation features

## Reporting Bugs

If you find a bug, please open an issue and include:

- What happened
- Expected behavior
- Steps to reproduce
